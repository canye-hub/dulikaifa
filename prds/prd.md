PRD（可直接开发版）
1. 产品概述

产品名称： 情绪表达翻译器（Emotion Translator） 

情绪表达翻译器

一句话定义：
将用户的情绪化表达，转化为更得体、更有效的沟通表达。

2. 产品目标

降低用户表达冲突风险

提升沟通质量

提供即时可用的表达替代方案

3. 用户与场景
核心用户

职场沟通人群

情侣 / 人际沟通人群 

情绪表达翻译器

使用场景

想发但不敢发的话

已写好但觉得“太冲”的话

不知道怎么表达更好

4. 功能范围（严格控制）

✅ 做：

输入一句话 → 输出3种表达

场景选择

调用 DeepSeek API

❌ 不做（必须遵守）：

用户登录

历史记录

多页面

复杂状态管理 

情绪表达翻译器

5. 核心功能说明
5.1 输入模块

功能：

用户输入文本

规则：

最大 200 字

支持多行输入

必填校验

5.2 场景选择

类型：下拉框

选项：

职场

恋爱

通用 

情绪表达翻译器

默认值：通用

5.3 生成按钮

行为：

点击 → 调用 API

按钮进入 loading 状态

状态变化：

默认：生成表达

loading：生成中...

5.4 输出模块

返回内容：

3条表达

语气要求：

委婉

直接

高情商 

情绪表达翻译器

5.5 错误处理
场景	提示
输入为空	请输入内容
超过200字	超出字数限制
API失败	生成失败，请重试
6. API设计（给Cursor直接用）
接口
POST /api/generate
请求参数
{
  "text": "用户输入内容",
  "scene": "职场 | 恋爱 | 通用"
}
返回数据
{
  "results": [
    "表达方式1",
    "表达方式2",
    "表达方式3"
  ]
}
7. DeepSeek API调用规范（重点）
模型建议

使用：

deepseek-chat
Prompt（直接给Cursor）
你是一个沟通优化专家。

请根据用户输入，在指定场景下，将其改写为3种表达方式：

场景：{scene}

用户原话：
{text}

请输出：
1. 更委婉的表达
2. 更直接的表达
3. 高情商表达

要求：
- 保留原意
- 表达自然
- 每条一句话
- 不要解释
- 用中文输出
返回解析规则

Cursor需要：

按换行或编号拆分为3条

去掉“1. 2. 3.”（如果有）
